Controlling who can see the Vynix widget

Updated 21 June 2026

You can control who sees the Vynix widget by only loading it for the users, pages, or environments you choose. The safest approach is to gate the widget in your own application code before the Vynix script is added to the page.

  1. Decide who should have access. Common options include admins, developers, QA testers, signed-in users, or users in a feature flag group.
  2. Add a condition in your app before loading the Vynix widget. For example, check the current environment, user role, route, or feature flag.
  3. Only insert or initialize the Vynix widget when the condition is true. If the condition is false, do not load the widget script.
  4. Test with both allowed and blocked users to confirm the widget appears only for the right audience.
  5. Review the Vynix docs for the current installation options and any workspace-level visibility settings.

Vynix is designed to sit on your site and capture developer context when someone clicks an issue, including the selected element, screenshot, console and network context, and an AI diagnosis. Because of that, you should avoid showing it to every visitor unless that is intentional.

The most common setup is to load Vynix only in development, staging, or preview environments. If you need it in production, restrict it to trusted users such as your team, authenticated testers, or users behind an internal feature flag.

Visibility should usually be controlled where you already manage access in your app. For example, use your authentication state, role checks, route guards, environment variables, or feature flag service to decide whether the widget should load.

If you remove access for a user, make sure the page no longer initializes the widget for that user. Hiding the button with CSS is not enough, because the widget may still be present in the page.

Related questions

Can I show the Vynix widget only to my team?

Yes. Gate the widget behind your app’s sign-in and role or permission checks, then load it only for approved users. Check the Vynix docs for the latest setup options.

Is it safe to just hide the widget with CSS?

No. Hiding the widget visually may still leave it loaded on the page. For stronger control, prevent the widget script from loading unless the user is allowed to see it.
