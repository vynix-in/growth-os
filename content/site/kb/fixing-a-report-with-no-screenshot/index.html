Fixing a report with no screenshot

Updated 21 June 2026

A Vynix report usually includes a screenshot of the page at the moment you annotate an issue. If the screenshot is missing, it is usually caused by browser permissions, page restrictions, or timing during capture.

  1. Reload the page, open the Vynix widget again, and create a new report. This fixes temporary capture or loading issues.
  2. Check that the page is fully loaded before clicking the issue. Screenshots may fail if the page is still navigating, rendering, or blocked by a loading overlay.
  3. Review browser privacy settings and extensions. Ad blockers, script blockers, strict tracking protection, or privacy extensions can prevent screenshot capture.
  4. Try the report in another supported browser or in a clean browser profile. This helps confirm whether the issue is caused by a browser setting or extension.
  5. Check whether the page uses protected content, cross-origin frames, browser-restricted pages, or security policies. Some areas cannot be captured by browser-based tools.
  6. If screenshots are still missing, include the console and network context from the report when contacting support, and check the latest setup guidance in the Vynix docs.

There may be no screenshot on your report if Vynix could not access the visible page content at capture time. Common causes include browser permission limits, security settings, content inside restricted frames, or extensions that block page scripts.

The rest of the report may still be useful. Vynix can still capture developer context such as the selected element, console details, network activity, and an AI diagnosis when those signals are available.

For setup requirements and the latest browser guidance, see the Vynix documentation at https://vynix.in/docs.

Related questions

Why does the report have console or network details but no screenshot?

Screenshot capture and developer-context capture use different browser capabilities. A page may allow Vynix to collect console or network context while still preventing a visual screenshot.

Can I still use a report without a screenshot?

Yes. The report can still help developers by showing the selected element, console output, network context, and AI diagnosis. If the missing screenshot is important, recreate the report after checking the steps above.
