Turning a Vynix note into a GitHub issue

Updated 21 June 2026

You can turn a Vynix note into a GitHub issue so your team or coding agent has the context needed to start fixing the problem. Vynix includes the selected element, screenshot, console and network context, and an AI diagnosis where available.

  1. Open the site where the Vynix widget is installed.
  2. Click the Vynix widget, then select the page element or area that has the issue.
  3. Review the note details, including the screenshot, captured context, and AI diagnosis.
  4. Choose the option to open or create a GitHub issue.
  5. Review the generated issue title and description, then edit anything you want to clarify.
  6. Select the repository and assignee if prompted, then create the issue.

A Vynix note is designed to carry developer-ready context into your workflow. When you create a GitHub issue from a note, Vynix can include the visual selection, relevant page details, console and network information, and a suggested diagnosis to help reduce back-and-forth.

If you do not see the GitHub issue option, check that your workspace has GitHub connected and that you have permission to create issues in the target repository. Setup details can vary, so refer to the Vynix documentation for the latest integration steps.

Before submitting the issue, quickly scan the generated content. Remove anything that should not be shared, add expected behavior, and include any steps to reproduce if they are not already clear from the captured context.

Related questions

What gets added to the GitHub issue?

The issue can include the Vynix note summary, screenshot, selected element context, console and network context, and an AI diagnosis. The exact fields may depend on your workspace settings and integration configuration.

Can I assign the issue to a coding agent?

Yes, if your GitHub workflow and Vynix setup support it, you can assign the new issue to a coding agent or another team member before creating it.
